# Описание проекта «Характеристики нелинейности и хаотичности»

## Назначение

Десктопное GUI-приложение для анализа временных рядов методами нелинейной динамики и теории хаоса. Выпускная квалификационная работа (ВКР). Приложение оценивает показатели нелинейности и хаотичности временного ряда на **R скользящих окнах шириной L со сдвигом d**.

**Репозиторий:** `git@github.com:evgenza/vkr.git`

---

## Реализованные методы анализа

| Метод | Файл | Что вычисляет |
|---|---|---|
| **Показатель Херста** | `Herst_f.py` | R/S-анализ — мера долговременной памяти ряда (H<0.5 — антиперсистентный, H=0.5 — случайный, H>0.5 — персистентный) |
| **Показатель Ляпунова** | `Lyapunov_f.py` | Чувствительность к начальным условиям (>0 = хаос, <0 = стабильность) |
| **Фрактальная размерность** | `fract_dim_f.py` | Корреляционная размерность — мера сложности аттрактора |
| **Информационный показатель** | `Inform_f.py` | Энтропия распределения — мера «эмерджентности» |
| **Показатель Гёльдера** | `Hoelder_f.py` | Локальная регулярность (гладкость) функции |
| **Тест Хинича** | `GLSTAT.py` | Проверка гауссовости (Shapiro-Wilk / Jarque-Bera) и линейности |
| **Сегментация** | `segment_my.py` | Обнаружение скачков скорости (через пороговую фильтрацию производной) |
| **Бифуркационный анализ** | `bifurk_my.py` | Фазовые портреты до/после скачка на плоскостях (Гёльдер × Фрактальная размерность) и (Гёльдер × Ляпунов) |
| **Кумулятивные суммы (CUSUM)** | `bifurk_my_1.py` | Обнаружение изменений статистических характеристик |
| **Куртозис и пик-фактор** | `Curth_my.py` | Динамика эксцесса и пик-фактора на скользящих окнах |
| **SSA-анализ** | `SSA_my.py` + `Huang_f.py` | Сингулярное спектральное разложение через SVD + огибающая Гильберта |
| **Спектрограмма** | `specgram_my.py` | 3D частотно-временное представление (scipy.signal.spectrogram) |
| **Двойное экспоненциальное сглаживание** | `exp_smooth_twice.py` | Сглаживание трендов (двухпроходный экспоненциальный фильтр) |

Интегральная формула фрактальной размерности с поправкой: **DT = (Dt + 4 − H + 1.21·I) / 3**, где Dt — корреляционная размерность, H — показатель Херста, I — информационный показатель.

## Интерфейс пользователя

Приложение имеет стилизованный GUI с градиентным фоном и цветовой кодировкой кнопок:

- **Заголовок** — градиент фиолетового цвета
- **Панель параметров** — 7 полей ввода: ширина окна L, начало отсчёта K, сдвиг d, число полос m, отклонение D, число окон R, коэффициент сглаживания α
- **Панель скачка** — номер скачка n (для бифуркационного анализа)
- **Кнопки анализа** (10 штук, сгруппированы по цвету):
  - Синий — Загрузка данных
  - Фиолетовый — Тест Хинича, Сегментация
  - Красный — Херст и Ляпунов, Показатель Гёльдера
  - Зелёный — Куртозис и пик-фактор
  - Оранжевый — Бифуркация, Спектрограмма
  - Бирюзовый — Кумулятивные суммы, Колебания (SSA)
  - Серый — Выход

Каждый анализ открывается в **отдельном модальном диалоге** (`PlotDialog`) со встроенной панелью Matplotlib (зум, перемещение, сохранение).

## Конфигурация

Централизованная через `config.py` — набор `dataclass`-ов:

- **`DefaultParameters`** — параметры по умолчанию (L=200, K=1, d=100, m=30, D=4.0, R=200, α=0.01)
- **`ValidationRules`** — диапазоны валидации каждого параметра
- **`SmoothingConfig`** — коэффициенты сглаживания по типам анализа (Херст=0.4, Гёльдер=0.2, бифуркация=0.1)
- **`AnalysisConstants`** — пороги и множители для алгоритмов
- **`PlotStyle` / `PlotSettings`** — стили графиков (размеры шрифтов, толщина линий)
- **`FileFormats`** — фильтры файлов для диалога

## Происхождение

Проект является **портом с MATLAB на Python**. Оригинальные `.m` файлы сохранены в папке `matlab/` — они являются прототипом, с которого велась разработка. Именование функций (`Herst_f`, `Hoelder_my`, `GLSTAT`, `bifurk_my`) сохранено от оригинала. Комментарии и индексация в Python-версии содержат пометки вида «MATLAB: y(n)».

## Сборка и запуск

```bash
make install          # Создание venv + установка зависимостей
make ui               # Генерация ui_form.py из form.ui
make run              # Запуск приложения
make build            # Сборка в standalone-бинарник через PyInstaller
make test             # Проверка работоспособности всех функций
make test-data        # Генерация синтетических тестовых данных
```

Бинарник собирается кросс-платформенно в `bin/{linux,macos,windows}/vkr_analysis`.

## Паттерны кодирования

- **Процедурный стиль** — основные функции анализа (`*_my.py`, `*_f.py`) — процедуры, принимающие массивы и параметры, возвращающие массивы
- **OOP-заготовка** — `base.py` содержит `BaseAnalyzer` / `WindowedAnalyzer` / `AnalysisResult` для будущего перехода на ООП, но текущий код ими **не пользуется**
- **Единый интерфейс фигур** — `setup_figure()` / `finalize_figure()` из `utils.py` позволяют функциям работать как standalone (plt.show), так и embedded (в Qt-диалоге)
- **Генератор окон** — `iter_windows()` — единый генератор скользящих окон
- **Шаблонный метод** — `_run_analysis()` в `MainWindow` — DRY-паттерн запуска любого анализа
